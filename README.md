# 🚒 제7회 대구 빅데이터 분석 경진대회 (4위)

> **대구시 소방차 출동로 취약지역 분석 및 머신러닝 기반 위험도 예측 및 정책 시뮬레이션**

---

## 🏆 대회 개요
- **대회명:** 제7회 대구 빅데이터 분석 경진대회  
- **주최:** 대구광역시 × 대구디지털혁신진흥원 / 데이콘  

### 📌 문제 정의
대구시 화재 대응 분석은 다음과 같은 한계를 가지고 있습니다:

- 출동시간의 높은 변동성 (시간/수요 영향)
- 지역별로 상이한 위험 요인
- 정책 효과 사전 검증의 어려움

이에 본 프로젝트는 다음을 목표로 합니다:

- 출동 도착시간 및 출동횟수 예측  
- 지역별 위험 요인 해석  
- 정책 개입 효과 정량 검증  

---

## 🎯 분석 접근

본 프로젝트는 단순 예측 모델 구축이 아니라,  
실제 정책 의사결정에 활용 가능한 구조를 만드는 것을 목표로 설계했습니다.

- 위험 지역 식별 → 예측
- 위험도 시각화 → 지도 기반 표현
- 위험 원인 분석 → SHAP 기반 해석
- 정책효과 검증 → 정책 시뮬레이션

이 네 가지 흐름을 하나의 파이프라인으로 연결하여  
예측 → 시각화 → 해석 → 정책 적용까지 연결된 End-to-End 의사결정 파이프라인을 구축했습니다.

## 🧩 데이터 처리

- 행정동 기준으로 다양한 출처의 데이터를 통합하여 분석 단위를 일관되게 구성  
- 결측치: 인구, 면적, 인구밀도 등 지역 특성을 반영한 가중 평균 방식으로 보정  
- 이상치: 도착시간 60분 이상 데이터 제거하여 모델 왜곡 방지  
- 범주형 변수: One-Hot Encoding 적용  
- 타깃 변수: log 변환 후 예측값은 역변환하여 해석 가능하도록 구성  

단순 전처리를 넘어, 각 변수의 의미와 관계를 고려하여  
데이터의 신뢰도를 높이는 방향으로 전처리를 설계했습니다.

## 🧠 분석 방법

### 🔹 데이터 분리 전략

화재 출동은 시간적 요인의 영향을 크게 받기 때문에  
하나의 통합 모델로 접근하기보다, 다음과 같이 데이터를 분리하여 분석했습니다.

- 시간별  
- 요일별  
- 월별  

각 단위별로 데이터 특성이 다르기 때문에  
독립적인 모델을 구성하여 보다 정밀한 패턴을 학습하도록 설계했습니다.

## 🗺️ 위험도 시각화

- 위험도 점수를 지도 기반으로 시각화하여 읍면동 단위 취약 지역 식별  
- 시간 / 요일 / 월 단위 위험도 분해를 통해 상황별 위험 패턴 분석  
- 위험도 점수는 출동횟수와 도착시간을 결합하여 산출  

예측 결과를 직관적인 형태로 표현하여  
분석 결과를 한눈에 파악할 수 있도록 구성했습니다.

이를 통해 운영 및 정책 대응에 활용 가능한 시각적 의사결정 기반을 마련했습니다.

---

## 🔍 SHAP 분석

- 지역별 주요 영향 변수 도출  
- 동일 위험지역 간 원인 차이 분석  
- 정책 개선 방향 도출  

예측 결과를 단순 수치로 끝내지 않고,  
왜 해당 지역이 위험한지 를 설명 가능한 형태로 확장했습니다.

이를 통해 지역별 맞춤형 정책 설계가 가능하도록 기반을 마련했습니다.

---

## 🧪 정책 시뮬레이션

- 도로폭, 협소도로 비율 등 정책적으로 개선 가능한 변수 조정  
- 변화 전 / 후 예측값 비교  

👉 단순 분석이 아닌  
정책 적용 시 효과를 정량적으로 검증하는 단계까지 확장

이를 통해 분석 결과를 실제 의사결정에 활용할 수 있는 형태로 연결했습니다.

---

## 📊 주요 결과

- 위험도 점수를 지도 기반 시각화로 표현하여 취약 지역을 직관적으로 식별할 수 있도록 구성 
- SHAP 분석을 통해 지역별 주요 영향 요인을 도출하고, 위험 발생 원인을 설명 가능하게 제시 
- 정책 시뮬레이션을 통해 지역별 위험 요인 변화에 따른 도착시간 개선 효과를 정량적으로 분석  
  (예: 다사읍 기준, 협소도로 개선 시 평균 도착시간 약 15.7% 단축 효과 확인)

## 📝 경진대회 참가 및 수상 후기

이번 프로젝트를 통해 가장 크게 느낀 점은  
모델보다 데이터가 더 중요하다는 것이었습니다.

이번 대회는 가공되지 않은 RAW 데이터를 기반으로 진행되었기 때문에,  
모델링 이전에 어떤 데이터를 선택하고 어떻게 가공할 것인지가  
전체 결과에 큰 영향을 미친다고 판단했습니다.

특히 필요한 데이터를 직접 확인하고 문제 정의에 맞는 데이터를 선별하고 통합하는 과정에서  
데이터 탐색과 전처리 능력의 중요성을 깊이 체감할 수 있었습니다.

또한 부족한 데이터는 공공 사이트를 통해 직접 수집하며,  
단순 분석을 넘어 데이터 수집부터 전처리, 모델링까지 이어지는 전체 과정을 경험할 수 있었습니다.

이 과정에서 데이터 선택 및 구조 설계 능력, 전처리 및 피처 엔지니어링 능력,  
그리고 문제 정의와 분석 방향 설정 능력을 함께 향상시킬 수 있었습니다.

아쉬운 점은 결과를 코드 수준에서 마무리한 점입니다.  
시간이 더 있었다면 대시보드나 프로그램 형태로 확장하여  
분석 결과를 보다 직관적으로 전달할 수 있는 구조로 발전시키고 싶었습니다.
